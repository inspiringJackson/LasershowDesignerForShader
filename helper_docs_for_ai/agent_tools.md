# 激光秀设计工具 Agent 可用工具清单 (Agent Tools)

为了引入 AI Agent 协助设计者完成激光秀编排工作，基于当前项目的核心架构（主要集中在 `src/core/commands.py`, `src/core/models.py` 以及 `src/ui/` 相关的组件），以下功能和操作可以被封装并集成为 Agent 可调用的工具（Tools）。

## 1. 工程与全局环境管理 (Project & Environment)
这些工具允许 Agent 初始化、配置和管理激光秀的基础环境。
* **新建/加载/保存工程**：允许 Agent 读写 `.lsd` 项目文件。
* **修改项目元数据**：如 BPM (节拍数)、Time Signature (拍号) 和总小节数。
* **导出/渲染操作**：调用导出模块，将编排好的动画导出为适用于 Minecraft 的 GLSL Shader 脚本。

## 2. 激光光源设备管理 (Laser Source Management)
Agent 可以在 3D 空间中增删和配置激光设备，进行舞台布光。
* **增删激光设备**：添加或移除激光光源 (对应 `AddItemCommand`, `RemoveItemCommand`)，同时添加操作可直接配置光源名称、类型和参数。
* **修改光源属性**：配置选中光源的参数 (对应 `PropertyChangeCommand`)：
  * **基础属性**：名称。
  * **空间属性**：位置 (`pos`)、方向 (`dir`)、法线/Up向量 (`localUp`)。
  * **视觉属性**：颜色 (`color`)、亮度 (`brightness`)、光束粗细 (`thickness`)。
  * **物理属性**：发散角 (`divergence`)、衰减系数 (`attenuation`)。
  * **其他参数**：参数 (`params`)。
* **光源类型切换**：设置单束、扇面、图案/SDF、粒子或实体扇面等类型。
* **主从阵列同步系统**：设置主控灯 (`is_master`) 和从属灯列表，配置偏移参数以生成阵列波浪或追光效果。

## 3. 时间轴与自动化轨道编排 (Timeline & Automation)
这是 Agent 进行自动编排和音乐卡点的核心工具集。
* **管理自动化轨道**：为特定设备的特定属性（如 `pos.x`, `color`, `brightness`）创建或删除自动化轨道。
* **关键帧 CRUD 操作**：
  * 在特定的时间（拍子/秒）插入关键帧。
  * 删除选中的关键帧。
  * 修改关键帧的时间位置和数值 (对应 `KeyframeMoveCommand`, `KeyframeValueChangeCommand` 等)。
* **关键帧曲线与插值控制**：
  * 修改两个关键帧之间的插值类型 (CurveType)：如 `LINEAR` (线性), `SMOOTH` (平滑), `HOLD` (保持), `STAIRS` (阶梯), `PULSE` (脉冲), `WAVE` (波浪) 等。
  * 调节曲线张力 (Tension) 实现 Ease-in/Ease-out 缓动效果。
* **批量生成与随机化**：在特定时间段内自动生成随机运动或闪烁序列的关键帧。

## 4. 媒体与音频同步 (Media & Audio)
用于 Agent 分析音乐节拍和对齐音频。
* **管理音频序列**：添加、移除背景音乐文件。
* **音频对齐与偏移**：调整音频在时间轴上的起始点、偏移量和裁剪长度 (对应 `SequenceResizeCommand` 等)。

## 5. 播放控制与预览反馈 (Playback & Preview)
Agent 可用于在编排过程中控制模拟器并获取反馈。
* **播放状态控制**：播放、暂停、停止。
* **时间轴跳转 (Seek)**：跳转到指定节拍或时间点。
* **相机与视口控制**：控制 3D 预览相机的视角（经纬度、距离、目标点），便于 Agent 多角度“观察”或评估效果。

---

### 给 Agent 工具开发者的设计建议
1. **坐标系与参数范围约束**：
   * 空间坐标 (`pos`) 通常限制在 `-3000` 到 `3000` 之间。
   * 颜色通道为 `0-255`。
   * 亮度 (`brightness`) 范围一般为 `0.0` 到 `10.0`。
   * 发散角 (`divergence`) 范围为 `0.0` 到 `180.0` 度。
2. **Undo/Redo 兼容性**：Agent 调用的所有修改工程状态的工具（特别是属性修改、关键帧增删改）必须通过内部的 `QUndoCommand` 包装系统执行，以保证设计者可以随时撤销 Agent 的操作，防止 Agent 误操作破坏工程。
3. **节拍吸附 (Snap)**：Agent 计算时间时，建议使用项目设定的量化网格（如 1/4 拍、1/8 拍）来生成关键帧时间，以确保动作与音乐卡点准确。
